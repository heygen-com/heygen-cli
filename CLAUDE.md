# HeyGen CLI

Go CLI wrapping HeyGen's v3 API. Built on Cobra, command surface auto-generated from OpenAPI spec.

## Build & Test

```bash
make build    # → bin/heygen
make test     # all tests are mocked, no API key needed
make lint     # golangci-lint
make clean
```

Smoke test against real API: `HEYGEN_API_KEY=<key> ./bin/heygen video list --limit 2`

## Project Layout

```
cmd/heygen/        Entry point, Cobra commands, hand-written enhancements
internal/errors/   CLIError, APIError, exit codes (0/1/2/3)
internal/auth/     auth.Resolver interface + implementations
internal/config/   config.Provider interface + implementations
internal/client/   HTTP client, RequestSpec, Executor
internal/output/   output.Formatter interface + implementations
gen/               GENERATED — do not hand-edit
codegen/           OpenAPI → Go codegen pipeline
```

## Go Standards

### Errors
- All command errors must be `*CLIError`. Use `clierrors.New()` (exit 1), `clierrors.NewAuth()` (exit 3), `clierrors.NewUsage()` (exit 2).
- API responses: `clierrors.FromAPIError(statusCode, apiErr, requestID)`.
- Flag/arg validation: always `NewUsage()` so exit code is 2, not 1.
- All error output goes through `formatter.Error()` as a JSON envelope on stderr. Never `fmt.Fprintln(os.Stderr, ...)`.

### Naming
- Packages: lowercase single word (`client`, `auth`, `config`).
- Interfaces get their own file (`resolver.go`, `formatter.go`). Implementations named by backing (`env_resolver.go`, `json_formatter.go`).
- Tests live next to their code: `foo_test.go` in the same package.

### Imports
- Group: stdlib, blank line, external, blank line, internal.
- Alias internal errors: `clierrors "github.com/heygen-com/heygen-cli/internal/errors"`.

### Testing
- All automated tests use `httptest.Server`. No real API calls in tests or CI.
- Command tests: use `runCommand()` in `cmd/heygen/testutil_test.go`. It creates a fresh Cobra tree, captures stdout/stderr/exit code, and renders errors through the formatter (same path as `main()`).
- Use `t.Setenv()` for env vars (auto-restored).
- Assert on exit codes (0/1/2/3) **and** stderr JSON envelope shape, not just error presence.
- Naming: `Test<Command>_<Scenario>` (e.g. `TestVideoList_AuthMissing`).

### Separation of concerns
- **`auth.Resolver`** is the single source of truth for credentials. `config.Provider` does not own API keys.
- **`config.Provider`** owns non-secret settings (`BaseURL`, output format, etc.).
- **`RequestSpec`** is the contract between commands and the executor. Commands produce a spec; the framework executes it. Commands never call the HTTP client directly.
- Files in `gen/` are deleted and regenerated by `make generate`. Never put hand-written logic there.
- Hand-written enhancements (poll configs, flag constraints, custom commands) go in `cmd/`.
